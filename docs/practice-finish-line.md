# 실습 최종 목적지

이 프로젝트의 목적은 큰 서비스를 만드는 것이 아니라, 작은 웹앱을 통해 Codex 기반 AI 페어 프로그래밍 운영 흐름을 끝까지 경험하는 것이다.

## 최종 제품 상태

Habit Check는 다음을 만족하면 실습용 완성 상태로 본다.

- 습관을 추가하고 삭제할 수 있다.
- 오늘의 습관 완료 상태를 저장할 수 있다.
- 완료된 습관을 숨길 수 있다.
- 날짜별 기록을 탐색할 수 있다.
- 데이터는 브라우저 `localStorage`에 저장된다.
- 정적 사이트로 GitHub Pages에 배포된다.
- Docker로도 실행할 수 있다.

## 최종 운영 상태

프로젝트 운영 흐름은 다음을 만족하면 완성 상태로 본다.

- GitHub Issue로 작업을 시작한다.
- Issue에는 수용 기준, 검증 계획, agent 라우팅이 포함된다.
- 기능 브랜치에서 변경한다.
- PR에는 변경 요약, 테스트 결과, 수동 확인, agent handoff가 포함된다.
- CI가 PR에서 통과한다.
- `main`은 ruleset으로 보호된다.
- README 진행 항목은 작업 종료 시 갱신된다.

## 실습 종료 기준

이 실습은 `Issue #1 날짜별 기록 보기`가 완료되면 종료한다.

완료 조건:

- `Issue #1`의 수용 기준을 모두 만족한다.
- 날짜별 기록 보기 기능이 구현된다.
- `Core and smoke tests`가 통과한다.
- 데스크톱과 모바일 화면에서 수동 확인한다.
- 기능 PR이 `main`에 merge된다.
- GitHub Pages 배포 URL에서 동작을 확인한다.
- README 진행 항목에 최종 완료가 기록된다.
- `Issue #1`을 close한다.

## 실습 이후 확장 후보

아래 항목은 이번 실습의 필수 종료 조건이 아니다.

- 데이터 내보내기/가져오기
- 주간 통계 또는 streak
- 다크 모드
- 서버 저장소
- 사용자 계정
- 실제 다중 agent 백엔드
